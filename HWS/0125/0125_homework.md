## 1. Built-in 함수와 메서드

.sort 예시

```python
a = [1, 3, 4, 5, 2, 7, 3]
a.sort()
print(a)
[1, 2, 3, 3, 4, 5, 7]
```

- `.sort`는 기존의 값을 변경한다.

sorted() 예시

```python
a = [1, 3, 4, 5, 2, 7, 3]
result = sorted(a)

print(result)
print(a)
[1, 2, 3, 3, 4, 5, 7]
[1, 3, 4, 5, 2, 7, 3]
```

- `sorted()`는 기존의 값을 두고 새로운 리스트로 변환한다.

## 2. .extend()와 .append()

.extend() 예시

```python
>>> a = [1,2,3]
>>> a.extend([4,5])
>>> a
[1, 2, 3, 4, 5]
>>> b = [6, 7]
>>> a.extend(b)
>>> a
[1, 2, 3, 4, 5, 6, 7]
```

- `.extend(x)` 함수에서 x에는 리스트만 올 수 있으며 원래의 a 리스트에 x 리스트를 더하게 된다.

.append() 예시

```python
>>> a = [1, 2, 3]
>>> a.append(4)
>>> a
[1, 2, 3, 4]
```

리스트의 맨 마지막에 x를 추가하는 함수이다. 리스트 안에는 어떤 자료형도 추가할 수 있다.

```python
>>> a.append([5,6])
>>> a
[1, 2, 3, 4, [5, 6]]
```

위와 같이 리스트에 리스트도 추가할 수 있다.

- 즉, append는 x 그 자체를 원소로 넣고 extend는 iterable의 각 항목들을 넣습니다.

## 3. 복사가 잘 된 건가?

```python
[1, 2, 5, 4, 5]
[1, 2, 5, 4, 5]
```

서로 같다. 대입 연산자(=)를 통한 복사는 해당 객체에 대한 객체 참조를 복사하기 때문이다. 해당 주소의 일부 값을 변경하는 경우 이를 참조하는 모든 변수에 영향을 주게 된다. 이를 해결하기 위해서는 얕은 복사(shallow copy) 또는 깊은 복사(deep copy)를 사용할 수 있다.
